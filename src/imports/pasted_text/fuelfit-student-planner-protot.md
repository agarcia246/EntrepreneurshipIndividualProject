Design a fully connected, high-fidelity mobile app prototype called “FuelFit Student Planner” for university students who go to the gym and want to eat healthier with minimal planning stress.

Goal of the app:
Help busy students plan workouts, basic healthy meals, and daily habits in one simple place. The app should feel realistic, polished, and startup-ready, but still student-friendly and practical.

Platform:
Mobile-first app prototype
iPhone 15 / modern smartphone frame
Clean component-based design system
Clickable prototype with working navigation between all key screens

Core concept:
This is not a hardcore fitness app. It is a simple student wellness planner focused on:
- weekly gym planning
- healthy meal planning
- habit tracking
- low-friction setup
- progress visibility
- simplicity over complexity

Target user:
University student
Busy schedule
Interested in gym, healthy eating, self-improvement
Often struggles with consistency, planning, and motivation
Does not want a complicated app

Brand and visual style:
- Modern, minimal, energetic
- Color palette: deep green, white, charcoal, muted blue accents
- Rounded cards
- Clean dashboard layout
- Strong spacing and typography
- Friendly but not childish
- Slight tech + wellness feel
- Use icons for meals, workouts, habits, progress, and planning
- Make it look like a real startup MVP

Build these screens and connect them with clickable flows:

1. Splash Screen
- App logo: FuelFit Student Planner
- Tagline: Plan your week. Train better. Eat smarter.
- Clean branded entry screen

2. Onboarding Flow (3 screens)
Screen 1:
- Welcome message
- Explain app value in one sentence
- CTA: Get Started

Screen 2:
- Ask user goals:
  - Build muscle
  - Stay consistent
  - Eat healthier
  - Lose fat
  - Improve routine
- Multi-select cards

Screen 3:
- Ask user preferences:
  - Workout days per week
  - Diet style
  - Time available for cooking
  - Notifications on/off
- CTA: Create My Plan

3. Sign Up / Log In
- Email and password
- Continue with Google
- Continue as guest
- Clean and realistic UI

4. Home Dashboard
Main dashboard with:
- Welcome back, Alex
- Weekly consistency score
- Today’s workout card
- Today’s meal plan summary
- Habit tracker summary
- CTA buttons:
  - View Weekly Plan
  - Log Workout
  - Log Meals
  - Update Habits
- Progress ring or bar
- Motivational but subtle microcopy

5. Weekly Planner Screen
- Weekly calendar layout
- Each day shows:
  - workout plan
  - meals
  - habits
- Ability to tap each day
- Simple drag/edit style cards
- CTA: Auto-generate week
- CTA: Edit manually

6. Workout Plan Screen
- List of workout days
- Each workout includes:
  - title
  - muscle group
  - duration
  - exercises
- Example plans:
  - Push Day
  - Pull Day
  - Legs
  - Full Body
- Exercise detail cards with reps and sets
- Button: Mark as completed
- Button: Swap workout

7. Meal Plan Screen
- Daily meal plan cards:
  - breakfast
  - lunch
  - dinner
  - snack
- Each meal should be simple and student-realistic
- Show estimated calories and protein
- CTA: Swap meal
- CTA: Save meal
- CTA: Generate easier option
- Keep meals practical, affordable, and healthy

8. Habit Tracker Screen
Track habits such as:
- Water intake
- Sleep 7+ hours
- Protein target
- Steps goal
- No junk food binge
- Gym session complete
Use toggles, checkboxes, and progress bars
Show streaks and weekly completion rate

9. Progress Screen
- Weekly completion stats
- Workouts completed
- Meals followed
- Habits completed
- Streak counter
- Small charts
- Reflection card:
  - What went well this week?
  - What was difficult?
- Keep visuals clean and motivational

10. Feedback / Check-In Screen
A simple weekly review screen where the user answers:
- How consistent did you feel this week?
- What got in the way?
- What should next week focus on?
- Rate your week from 1–5
This should feel reflective and helpful

11. Settings / Profile
- Name
- Goal
- Notification preferences
- Dietary preferences
- Workout frequency
- Edit profile
- Dark mode toggle
- Sign out

12. Empty States
Create realistic empty states for:
- no weekly plan yet
- no workouts logged
- no meals added
- no habits tracked
Each should include a friendly CTA

13. Notification / Reminder UI
Create sample reminder cards for:
- Workout in 1 hour
- Meal prep reminder
- Log today’s habits
- Weekly review available

14. Quick Add Modal / Floating Action Button
Floating action button from dashboard
Lets user quickly:
- log a workout
- add a meal
- check off habits
- update weekly plan

Prototype interactions:
- Make all bottom navigation tabs clickable
- Use bottom nav with:
  - Home
  - Plan
  - Track
  - Progress
  - Profile
- Connect onboarding to sign-up to dashboard
- Connect dashboard cards to their detailed screens
- Connect meal swap and workout completion interactions
- Connect habit toggles and progress visuals
- Create basic realistic transitions

Important UX rules:
- Prioritize simplicity
- Avoid clutter
- Keep actions obvious
- Make setup fast
- Reduce user effort
- Design for students with limited time
- The app should feel like it removes friction, not adds more

Generate realistic content:
- Use believable workout examples
- Use practical healthy meal examples for students
- Use realistic stats and progress examples
- Use short, clear, natural app copy

Also create:
- Reusable components
- Buttons, cards, nav bars, form fields, progress widgets
- Consistent spacing and design system styles
- Mobile prototype with polished UI ready for presentation

Final output should feel like:
A real, launchable MVP prototype for a student fitness + healthy eating planning app, built in Figma with connected screens and a strong modern UI.
